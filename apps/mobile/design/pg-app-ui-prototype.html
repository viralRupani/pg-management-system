Component & style reference

The reusable patterns behind every screen. Colors are CSS variables — swap --accent per PG; the semantic status palette stays fixed so meaning is consistent across brands.

Accent (white-label)

--accent  (per-PG)
soft · soft2 · base · press · deep — all derived via color-mix()
Tints/shades are computed from one variable, so any accent works without hand-picking shades.

Neutrals & text

Ink (primary) #111827
Ink-2 (muted) #6b7280
Ink-3 (hint) #9ca3af
Line #e9ebef
Page bg #f3f4f6
Surface #ffffff

Status badges — fixed semantics

Pending Submitted In progress Paid Verified Resolved Overdue Rejected Not started
Amber — awaiting action (Pending)
Blue — in motion (Submitted / In progress)
Green — success (Paid / Verified / Resolved)
Red — needs attention (Overdue / Rejected)
Soft tinted fills (not solid) so badges read clearly beside any brand accent and never clash with it.

Buttons

Min height 48px for comfortable tap targets. Full-width primary at the bottom of forms/sheets.
